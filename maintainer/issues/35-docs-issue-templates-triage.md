# [35] docs(repo): add issue templates and maintainer triage guide

## Summary
Standardize incoming issue quality and maintainer triage process.

## Problem
Without templates, issue quality varies and slows contributor onboarding.

## Scope
- Add bug/feature/docs templates.
- Add label and triage guide for maintainers.

## Files
- `.github/ISSUE_TEMPLATE/*`
- `.github/labels.md`
- `README.md` (contribution section)

## Acceptance Criteria
- Templates enforce reproducibility and acceptance criteria fields.
- Maintainer triage flow is documented.
- Contributor guidance references templates.

## Labels
`documentation` `devex` `maintainers`

## How To Test
### Verify process
1. Open "new issue" flow and confirm templates appear.
2. Create sample issues for each template.
3. Confirm required sections are present and useful for triage.
4. Validate contribution docs link to templates.
